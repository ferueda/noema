# Contributing to Noema

Use this directory when changing Noema itself. The root `README.md` introduces
the product and its current commands. These guides explain how humans and agents
prepare, test, and hand off repository changes.

## Start here

- [Project intent](../project-intent.md) — purpose, priorities, hard boundaries,
  and unsafe assumptions.
- [Architecture](../architecture.md) — accepted components, ownership, data flow,
  and core model.
- [Roadmap](../roadmap.md) — milestone order, completion gates, and deferred
  work.
- [Development](development.md) — prerequisites, setup, commands, mutability, and
  failure flow.
- [Testing](testing.md) — proof layers, test placement, fixtures, and verification
  commands.
